F2L

In this step, we will use <R,U> to expand our 2x2x3 into a 2x3x3. Generally, this is done by first solving a 2x2x1 square, and then attaching a 2x2x1 pair.

As a general rule of thumb, this step should take no more than 16 moves in the worst case (8 for square, 8 for pair). While pretty straightforward, there are some tricks that can be used to increase efficiency. On the examples page, the following solves contain interesting F2Ls that you might find helpful:

1 forms orange-green-white pair and preserves it while solving the red-green-white square instead of greedily solving it right away
3 U2 R2 U' R forms orange-green-white and white-green-red pairs, which are then solved into a block together
4 constructs two 2x3x1 lines and solves them together (in this case, more efficient than solving a square and a pair)
5 instead of solving the F2L on white, we use the free yellow-orange-blue pair staring us in the face to build the F2L on blue
7 once again we can use the free square right in front of us, giving a very lucky F2L
10 instead of solving F2L on white, we use the yellow-green-orange pair staring us in the face to build pseudo-F2L on yellow
16 instead of inserting the last pair with U R U' R', we use U2 R U2' R' to force a Sune/Antisune last layer case (these tend to be ~3-4 moves shorter)
44 after inserting the first line in Block, the second line is very awkward. Instead, we can make the right half of the F2L right away and complete the second line afterwards.
48 since we solved the first square while solving the Block, we can place the last corner + complete corner orientation with one of 27 algorithms (CLS), leaving only EPLL.
